<objective>
## Goal
Build the BASE MCP server with 7 generic surface CRUD tools, add surface registration schema to workspace.json, and register the server in .mcp.json.

## Purpose
This is the core infrastructure that makes data surfaces usable. Without this server, surfaces are just JSON files with hooks — no way to read, write, search, or manage them through MCP tools. The BASE MCP operates on ANY registered surface through a uniform interface, unlike CARL MCP's purpose-built tools.

## Output
- `.base/base-mcp/` — Complete MCP server with index.js, package.json, tools/surfaces.js
- 7 tools: base_list_surfaces, base_get_surface, base_get_item, base_add_item, base_update_item, base_archive_item, base_search
- workspace.json updated with `surfaces` section (empty, ready for Phase 4 registrations)
- .mcp.json updated with base-mcp server entry
- Source package mirror at .claude/skills/base/packages/base-mcp/
</objective>

    ```json
      "name": "base-mcp",

      "main": "index.js",

      }
    }
    ```

    **2. Create .base/base-mcp/index.js:**
    Follow carl-mcp/index.js pattern exactly:
    - Same imports: Server, StdioServerTransport, CallToolRequestSchema, ListToolsRequestSchema, path, fileURLToPath
    - WORKSPACE_PATH resolves from __dirname: `path.resolve(__dirname, '../..')`  (base-mcp/ → .base/ → workspace root)
    - Import TOOLS and handleTool from './tools/surfaces.js'
    - Server name: "base-mcp", version: "1.0.0"
    - Build TOOL_HANDLERS lookup from surfaceTools
    - debugLog prefix: '[BASE]'
    - Same request handlers (ListTools, CallTool) as carl-mcp
    - Same error handling pattern
    - Same runServer() and startup

    **3. Create .base/base-mcp/tools/surfaces.js:**
    Follow carl-mcp/tools/psmm.js pattern for structure:
    - Import readFileSync, writeFileSync, existsSync from 'fs'
    - Import join from 'path'
    - debugLog prefix: '[BASE:surfaces]'

    **Helper functions:**
    - `getWorkspaceJson(workspacePath)` — reads workspace.json, returns parsed object
    - `getSurfaceConfig(workspacePath, surface)` — gets a surface's registration from workspace.json surfaces section. Returns null if not found.
    - `readSurface(workspacePath, surface)` — reads .base/data/{file} for a surface, returns parsed JSON. Uses the `file` field from surface registration.
    - `writeSurface(workspacePath, surface, data)` — writes surface JSON with updated last_modified timestamp
    - `generateId(prefix, items)` — generates next ID: {prefix}-{NNN} where NNN is zero-padded max existing + 1
    - `formatTimestamp()` — ISO timestamp with timezone

    **7 tools (export as TOOLS array, export handleTool function):**

    1. `base_list_surfaces` — No params. Reads workspace.json surfaces. For each surface, counts items in its data file. Returns array: [{name, description, file, items_count, hook}].

    2. `base_get_surface` — Params: `surface` (string, required). Reads the surface's JSON file. Returns the full surface object (metadata + items). Error if surface not registered.

    3. `base_get_item` — Params: `surface` (string, required), `id` (string, required). Reads surface, finds item by id. Returns the item object. Error if surface not registered or item not found.

    4. `base_add_item` — Params: `surface` (string, required), `data` (object, required). Reads surface config for schema. Validates required_fields from schema if present. Auto-generates ID using id_prefix from schema. Adds `added: timestamp` field. Appends to items array. Writes file. Returns the new item with its generated ID.

    5. `base_update_item` — Params: `surface` (string, required), `id` (string, required), `data` (object, required). Finds item by id. Merges data fields onto existing item (shallow merge, does NOT replace entire item). Adds `updated: timestamp`. Writes file. Returns updated item. Error if not found.

    6. `base_archive_item` — Params: `surface` (string, required), `id` (string, required). Finds item in items array. Removes from items. Adds `archived: timestamp` to item. Appends to archived array. Writes file. Returns archived item. Error if not found.

    7. `base_search` — Params: `query` (string, required), `surface` (string, optional). If surface specified, search that surface only. If not, search all registered surfaces. Search: case-insensitive substring match across all string values in each item. Returns array: [{surface, id, title, match_field}].

    **Tool inputSchema format (MCP standard):**
    Each tool in TOOLS array has: `{ name, description, inputSchema: { type: "object", properties: {...}, required: [...] } }`

    **handleTool(name, args, workspacePath):**
    Switch on tool name, call appropriate function, return result object. Return null for unrecognized tools (matches carl-mcp pattern).

    Avoid: Importing anything beyond fs, path
    Avoid: Making network calls
    Avoid: Complex validation — just check required_fields from schema if present
    Avoid: Writing to workspace.json from tools — tools only read config, write data files
  </action>
