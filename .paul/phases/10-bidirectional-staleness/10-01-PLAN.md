<objective>
## Goal
On every session start, read `timestamps.updated_at` from each satellite's paul.json and write a `last_activity` field to its workspace.json entry. The groom_satellites health step then reads this structured field instead of parsing free-text from STATE.md.

## Purpose
Phase 9's groom_satellites step parses "Last activity" text from STATE.md — fragile, locale-dependent, breaks if the format drifts. paul.json has a machine-readable ISO timestamp (`timestamps.updated_at`) that reflects the last time PAUL wrote to the project. Using that as the staleness source makes health checks deterministic.

## Output
- `satellite-detection.py` — refreshes `last_activity` on all registered satellites every session, in addition to registering new ones
- `groom.md` — groom_satellites uses `last_activity` from workspace.json (fallback: STATE.md parse if field absent)
- `apps/base/src/hooks/satellite-detection.py` — source sync
- `apps/base/src/tasks/groom.md` — source sync
</objective>
