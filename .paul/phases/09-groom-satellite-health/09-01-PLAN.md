<objective>
## Goal
Add satellite health checking to the BASE groom cycle. Each satellite in workspace.json gets a `groom_check` boolean (default: true). During groom, enabled satellites are inspected — their STATE.md is read and evaluated against health criteria. Issues are surfaced in the groom report.

## Purpose
Groom currently has no visibility into PAUL satellite projects. A workspace can have 8+ satellites and none of them get checked unless the user manually opens each one. Health drift goes undetected.

## Output
- `workspace.json` — all satellite entries gain `groom_check: true`
- `groom.md` — new `groom_satellites` step added after `groom_directories`
- `apps/base/src/tasks/groom.md` — source sync
</objective>

       b. Parse "Last activity" timestamp and "Loop Position" section
       c. Evaluate health:
          - STUCK LOOP: Loop position shows PLAN ✓ APPLY ○ or PLAN ✓ APPLY ✓ UNIFY ○, and last activity > 7 days ago
          - ABANDONED PHASE: Last activity > 14 days ago and milestone not COMPLETE
          - MILESTONE DRIFT: All phases show COMPLETE but no new milestone has been started (loop shows ○ ○ ○)
    4. Collect issues across all satellites
    5. If issues found: surface as formatted list "⚠️ Satellite health issues:\n  - {satellite}: {issue}"
    6. If no issues: output single line "Satellites: all healthy ({N} checked)"

    Do NOT auto-fix issues — report only. Operator decides what to do.
    Do NOT read satellite files beyond STATE.md in this step.
  </action>
