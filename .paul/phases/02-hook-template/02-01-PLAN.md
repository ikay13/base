<task type="auto">
  <name>Task 1: Create the canonical hook template</name>
  <files>.base/hooks/_template.py</files>
  <action>
    Create `.base/hooks/_template.py` as a documented Python template that encodes the complete hook contract from BASE-V2-SPEC.md Component 2.

    **Structure (following psmm-injector.py pattern):**

    1. Shebang: `#!/usr/bin/env python3`
    2. Module docstring: Explain this is a TEMPLATE, not a runnable hook. Document the contract:
       - Reads ONE JSON file from .base/data/{SURFACE_NAME}.json
       - Outputs compact XML-tagged block to stdout
       - Wrapped in <{SURFACE_NAME}-awareness> tags
       - Includes BEHAVIOR directive block
       - Exits cleanly — never crashes, never blocks
    3. DO rules section (as comments):
       - Read the JSON file
       - Format compact summary (IDs, one-line descriptions, grouping by priority/status)
       - Include item count summaries
       - Include behavioral directive (passive by default)
       - Handle missing/empty/malformed files gracefully (output nothing, exit 0)
       - Use absolute paths via Path(__file__).resolve()
    4. DO NOT rules section (as comments):
       - Never write to any file
       - Never make network calls
       - Never import heavy dependencies
       - Never read multiple data files (one hook = one surface)
       - Never include full item details (that's what MCP tools are for)
       - Never include dynamic logic (time-based, session-count, etc.)
    5. Imports: sys, json, pathlib.Path only
    6. Path resolution: HOOK_DIR → WORKSPACE_ROOT → DATA_FILE using SURFACE_NAME placeholder
    7. main() function with:
       - stdin read for hook input (session_id extraction)
       - File existence check → exit 0 if missing
       - JSON parse with try/except → exit 0 if malformed
       - Items extraction with configurable key (default: "items")
       - Empty check → exit 0 if no items
       - Summary formatting section (marked with CUSTOMIZE comment)
       - Output block with:
         a. Opening tag: <{SURFACE_NAME}-awareness items="{count}">
         b. Summary lines
         c. BEHAVIOR directive block
         d. Closing tag
       - sys.exit(0)
    8. if __name__ == "__main__": main()

    **Placeholder convention:**
    Use `SURFACE_NAME` as the placeholder variable at the top of the file.
    Set it to `"example"` as default with a comment: `# CHANGE THIS: your surface name`

    **Behavioral directive (hardcoded in template):**
    ```
    BEHAVIOR: This context is PASSIVE AWARENESS ONLY.
    Do NOT proactively mention these items unless:
      - User explicitly asks (e.g., "what should I work on?", "what's next?")
      - A deadline is within 24 hours AND user hasn't acknowledged it this session
    For details on any item, use base_get_item("{surface}", id).
    ```

    Avoid: Making the template overly abstract — it should be a concrete, copy-and-customize file
    Avoid: Adding argparse or CLI flags — hooks receive input via stdin only
    Avoid: Using any imports beyond sys, json, pathlib
  </action>

    - grep "SURFACE_NAME" .base/hooks/_template.py returns multiple matches (placeholder present)
