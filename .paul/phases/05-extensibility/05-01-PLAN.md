<objective>
## Goal
Build the /base:surface commands (create, convert, list) as Claude Code skill task files so any user can create custom data surfaces without manual wiring.

## Purpose
Phases 1-4 built the infrastructure and proved the pattern with active + backlog. Phase 5 makes that pattern accessible — users run `/base:surface create` and get a guided workflow that generates the JSON file, hook, workspace.json registration, and settings.json entry automatically.

## Output
- `.claude/skills/base/tasks/surface-create.md` — Guided surface creation workflow
- `.claude/skills/base/tasks/surface-convert.md` — Convert @-mentioned markdown to surface
- `.claude/skills/base/tasks/surface-list.md` — Show registered surfaces with status
- Updated `base.md` entry point with new command routing
</objective>

## AC-1: surface-create.md implements guided 5-step workflow
```gherkin
Given a user runs /base:surface create {name}
When the skill task file is loaded
Then it guides through 5 steps: Define, Schema, Injection, Tools, Generate
And outputs: .base/data/{name}.json, .base/hooks/{name}-hook.py, workspace.json registration, settings.json hook entry
And the generated hook follows _template.py contract
And the user can create a surface end-to-end without manual file editing
```

## AC-2: surface-convert.md converts markdown to surface
```gherkin
Given a user runs /base:surface convert {file}
When the skill task file is loaded
Then it reads the markdown file and detects structure (sections, items, priorities)
And proposes a JSON schema based on detected patterns
And after user confirmation, generates the same outputs as create
And migrates markdown content into JSON items
And offers to remove @-reference from CLAUDE.md
```

```gherkin
Given a user runs /base:surface list
When the skill task file is loaded
Then it calls base_list_surfaces or reads workspace.json

```

  <name>Task 1: Create surface-create.md task file</name>
  <files>.claude/skills/base/tasks/surface-create.md</files>
  <action>
    Create a Claude Code skill task file following the scaffold.md pattern.

    **Structure:**
    - `<purpose>` — Guided creation of a new data surface
    - `<user-story>` — As an AI builder, I want to create custom surfaces...
    - `<when-to-use>` — /base:surface create, "create a surface", "add a new surface"
    - `<context>` — @.base/hooks/_template.py, @.base/workspace.json
    - `<steps>` — 5 steps matching spec:

    **Step 1: Define**
    - Extract surface name from args or ask
    - Ask: "What does this surface track?"
    - Validate name: lowercase, no spaces, not already registered
    - Set description

    **Step 2: Schema**
    - Ask: "What fields does each item need?"
    - Guide through: required fields, field types, priority/status enums, ID prefix
    - Default required_fields: ["title"] minimum
    - Default id_prefix: first 3 chars of name uppercase
    - Time rules optional (review_by thresholds)

    **Step 3: Injection**
    - Ask: "How should items appear in context?"
    - Grouping strategy: by priority (default), by status, by date, flat
    - Summary format: what fields to show per line
    - Behavioral mode: silent (default), proactive, threshold-based
    - Staleness thresholds per priority level

    **Step 4: Tools (informational)**
    - All 7 BASE MCP tools auto-work with any registered surface
    - No configuration needed — just inform user that base_get_item, base_add_item, etc. work automatically
    - Schema-driven validation via workspace.json

    **Step 5: Generate**
    - Create `.base/data/{name}.json` with empty envelope: { surface, version: 1, last_modified, items: [], archived: [] }
    - Generate `.base/hooks/{name}-hook.py` from _template.py customized with:
      - SURFACE_NAME = "{name}"
      - Grouping logic from Step 3
      - Summary format from Step 3
      - Staleness thresholds from Step 3
      - Behavioral directive with surface name
    - Add surface registration to workspace.json surfaces section
    - Add hook to .claude/settings.json UserPromptSubmit array
    - Report what was created

    **Key:** One question at a time. Wait for response. Build momentum.

    Avoid: Making the task file overly long — keep steps focused
    Avoid: Duplicating _template.py content inline — reference it via @
    Avoid: Hardcoding Chris-specific paths — use Path resolution patterns
